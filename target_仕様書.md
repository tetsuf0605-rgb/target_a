## 0. 概要

子どもが長期休み(夏休みなど)の目標を設定し、毎日自分で実績を記録して、進捗をグラフで確認できるアプリ。

- **単一 `index.html` 完結**。Vanilla JS のみ。外部ライブラリ・外部画像・外部API禁止。グラフは自前の SVG 描画。
- データの正本は **localStorage**(端末ごと保存でよい、と決定済み)。HTMLに焼き込む確定データはない。
- デザインは今回は**装飾なし・シンプルで見やすいだけ**にする(後で別途テーマを載せるため、CSSは差し替えやすいよう class 名を整理しておく)。
- 対象デバイス: iPad / iPhone / PC の Safari。タップ操作前提、ボタンは大きめ。文字は小学生が読めるよう、UI文言はひらがな多めにする。

## 1. データ構造(localStorage `target_v1` に単一JSONで保存)

```js
{
  version: 1,
  activePeriodId: "p_xxxx",   // 現在表示中の期間
  periods: [
    {
      id: "p_xxxx",           // "p_" + タイムスタンプ等で一意に
      name: "2026年 夏休み",
      start: "2026-07-18",    // YYYY-MM-DD
      end: "2026-08-31"
    }
  ],
  goals: [
    // 共通フィールド
    // id: "g_xxxx", periodId, name, type, order(表示順), createdAt

    // type: "daily"(毎日固定)
    {
      id: "g_1", periodId: "p_xxxx", type: "daily",
      name: "あさ6じにおきる",
      mode: "check",          // "check"(○×式) または "number"(数値式)
      // mode:"number" のときのみ:
      target: 20,             // 1日の目標値(例: 腕立て20回)
      unit: "かい"
    },

    // type: "weekly"(週ごとに目標値が変わる。各週の値は手入力)
    {
      id: "g_2", periodId: "p_xxxx", type: "weekly",
      name: "なわとび",
      unit: "かい",
      weeks: [50, 60, 70, 80, 90, 100, 110]
      // weeks[0] = 期間開始日を含む第1週の1日あたり目標値。
      // 週は「期間開始日から7日ごと」に区切る(曜日基準にしない)。
      // 期間の週数より weeks が短い場合、最後の値を以降の週にも適用する。
    },

    // type: "total"(期間合計型)
    {
      id: "g_3", periodId: "p_xxxx", type: "total",
      name: "けいさんドリル",
      unit: "ページ",
      totalTarget: 80         // 期間中の合計目標
    }
  ],
  records: [
    // 1日×1目標につき最大1件。同じ日に再入力したら上書き(確認後)。
    { date: "2026-07-20", goalId: "g_1", value: 1 },
    // daily(check): value は 1(できた) / 0(できなかった)
    // daily(number), weekly, total: value は入力された数値
  ]
}
```

実装上の注意:
- 読み込み時に `version` を見て、将来のデータ移行に備えた分岐だけ用意しておく(今は v1 のみ)。
- JSONが壊れていた場合は初期状態で起動し、alert で「ほぞんデータをよみこめませんでした」と通知(勝手に上書き保存はせず、次の保存操作まで既存データを残す)。
- 保存は変更のたびに即時 `localStorage.setItem`。

## 2. 画面構成(下部タブで3画面切り替え)

### 2-1. きょう(記録画面) — 起動時の初期画面

- 上部に日付表示と `◀ / ▶` ボタン。初期値は今日。期間の範囲外には移動できない。
- **今日以外の日付を表示中は、画面上部に「◯月◯日のきろくをへんしゅうちゅう」と目立つ帯を出す。**
- アクティブ期間の目標を order 順に一覧表示。各目標カードの内容:
  - **daily(check)**: 大きな「できた!」ボタン。押すと value=1。もう一度押すと取り消し(0でなくレコード削除)。すでに記録がある状態から変更するときは確認ダイアログ(§4)。
  - **daily(number) / weekly**: 数値入力欄(`inputmode="numeric"`)+「きろく」ボタン。その日の目標値(weeklyは該当週の値)を「もくひょう: 60かい」のように併記。目標達成なら ✅ を表示。
  - **weekly** はカードに「だい◯しゅうめ」も表示。
  - **total**: 「きょうすすんだページすう」を入力。カード内に「ごうけい ◯/80 ページ」「のこり ◯ページ」を表示。
- 既に記録済みの日の値を変更・再入力するときは必ず確認ダイアログを挟む(§4)。新規入力(その日その目標の記録がまだ無い)は確認不要。
- 期間が1つも無い/目標が1つも無い場合は、「せってい」への誘導メッセージを表示。

### 2-2. グラフ(進捗画面)

すべて自前SVG。目標ごとにカードを並べる。

- **total型**: 折れ線グラフ。X軸=期間の日付、Y軸=累積ページ数。
  - 実績の累積線と、**理想ペース線**(開始日0 → 終了日totalTargetの直線)を重ねる。
  - 実績が理想より上なら「よていより はやい!」、下なら「あと◯ページで よていどおり!」の一言を添える。
- **daily型**: カレンダー形式(期間の全日をマス目で表示)。
  - check: 達成日に◯。number: 目標達成日に◯、記録あるが未達は△。
  - 現在の**連続達成日数**と最長連続記録を表示(「◯にち れんぞく!」)。
- **weekly型**: 週ごとの棒グラフ(その週の1日平均実績)+各週の目標値を横線で重ねる。あわせて日別の達成◯もカレンダー式で小さく表示。
- 期間全体のサマリーを最上部に1枚: 経過日数/全日数、全目標の本日達成率など簡単なもの。

### 2-3. せってい(管理画面)

- **期間の管理**: 一覧表示、追加(名前・開始日・終了日)、編集、削除、「この期間をひらく」(activePeriodId切り替え)。
  - 過去の期間もここから開けば記録・グラフを閲覧できる(将来の夏・冬休み対応)。
- **目標の管理**(アクティブ期間の目標): 一覧、追加、編集、削除、表示順の上下移動。
  - 追加/編集フォームはタイプ選択(daily/weekly/total)で入力欄が切り替わる。
  - dailyはさらに check / number を選択。
  - weeklyの週別目標値は、期間の日数から週数を自動計算して週数ぶんの入力欄を出す(手入力)。
- **記録をリセット**: 全データ削除。二重確認(§4)。

## 3. 週の計算・日付処理

- 週番号 = `floor((対象日 - 期間開始日) / 7日) + 1`。期間開始日からの経過日数ベース。
- 日付はすべて `YYYY-MM-DD` の文字列で保持し、比較も文字列で行う(タイムゾーン事故防止)。今日の日付はローカル時刻から生成。

## 4. 確認ダイアログ(重要・決定事項)

以下の操作では必ず `confirm()` 相当の確認を挟む(見た目は自前のモーダルでよい):

| 操作 | 文言例 |
|---|---|
| 記録済みの値の変更・取り消し | 「◯月◯日の『なわとび』のきろくを 60かい → 70かい にかえます。いいですか?」 |
| 目標の編集の保存 | 「もくひょう『けいさんドリル』のせっていをかえます。いいですか?」 |
| 目標の削除 | 「もくひょう『◯◯』をけします。きろくもきえます。いいですか?」+ 削除時はその目標のrecordsも削除 |
| 期間の削除 | 「きかん『◯◯』をけします。もくひょうときろくもぜんぶきえます。いいですか?」 |
| 記録をリセット | 2段階: 「ぜんぶのきろくをけします。いいですか?」→「ほんとうにけしますか? もとにもどせません」 |

## 6. CSS方針(後のデザイン差し替えのため)

- 色・フォント・角丸などは CSS カスタムプロパティ(`--main-color` 等)に集約し、`:root` を差し替えるだけでテーマ変更できる構造にする。
- class 名は BEM 風に整理(`goal-card`, `goal-card__title` など)。
- 今回は白背景+1アクセント色程度のミニマルな見た目でよい。
